8-Phase Async Pipeline Celery Canvas

Celery chain() drives phases 1–3 and 8 on the pipeline queue. A group() | callback chord runs phases 4–7 in parallel on the analysis queue. Progress is polled by the client every 5 seconds.

Core Data Model ERD

PostgreSQL is the durable system of record. Redis holds transient task state only. S3/MinIO holds artifact blobs — keys (storage paths) are stored in PostgreSQL, blobs never are (AD-6).

Architecture Decision Summary

Twelve binding decisions. Each constrains a specific part of the codebase and prevents a named failure mode.

AD-1 · Modular monolith

No inter-app HTTP calls. All cross-app calls are direct Python imports from services.py or selectors.py. No localhost requests.

AD-2 · OrgScopedModel

Every org-owned model uses .for_org(org) queryset. API returns 404 for cross-org access (hides existence). Web UI returns 403 (better UX for shared links).

AD-3 · Service layer purity

Services accept and return plain Python objects only — no HttpRequest, no Celery Task instance. Same function callable from view and task.

AD-4 · Two Celery queues

pipeline queue: phases 1–3, 8, Beat cleanup. analysis queue: phases 4–7. Prevents vulnerability scans from starving new job submissions.

AD-5 · React SPA: REST API only

SPA served as static files via WhiteNoise. All data flows through /api/v1/. No Django template context passes business data to React.

AD-6 · Storage triad

PostgreSQL: durable models. Redis: task state keys and TTL-cached API responses. S3/MinIO: all artifact blobs. Blobs never written to PostgreSQL or Redis.

AD-7 · Per-org concurrency gate

Count PENDING|PROGRESS jobs before enqueue. If ≥ SBOM_MAX_CONCURRENT_JOBS_PER_ORG (default 5), return 429 with Retry-After. Owned by manifests/views.py.

AD-8 · AbstractAPIKey subclass

OrgApiKey extends AbstractAPIKey from djangorestframework-api-key. Library handles SHA-512 hashing, generation, DRF auth. ~20 lines of customisation.

AD-9 · Graph JSON shape

Graph endpoint returns Cytoscape.js-native {"nodes":[{"data":{…}}], "edges":[{"data":{…}}]}. No PyVis HTML. SVG download via separate presigned URL.

AD-10 · delay_on_commit()

Always task.delay_on_commit() from inside a DB transaction. Never plain .delay() from a view or signal. Prevents worker reading stale state.

AD-11 · Presigned URL downloads

Artifact downloads return 303 See Other to a 24h presigned S3/MinIO URL. Django never reads or streams artifact bytes.

AD-12 · SBOMJob.status ownership

SBOMJob.status written only by Celery task code via sbom/services.py. Views read status, never write it. Exception: initial PENDING at job creation.
